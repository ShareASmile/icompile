You Don't Need Jekyll!

Posted at Sat, 02 May 2020 23:21:10 GMT

Yes, GitHub Pages Are Static (But...)


Yes, GitHub pages are static, and served from a CDN (through Varnish).

Slightly before the content of your website is copied to the CDN,
GitHub pipes the content through Jekyll, which is a site-generator engine.

Seriously! What Jekyll Is Good For?


The most common way Jekyll is used, is to customize the generic GitHub website-template to your repository name and description, but it is also used to change markdown to HTML.
Advance users which include/edit their _config.yml in the root of the project can specify instructions to the Jekyll-engine for page-redirections and error-pages.

There are few pages that explain more about Jekyll and GitHub-pages: https://help.github.com/en/github/working-with-github-pages/about-github-pages-and-jekyll, https://help.github.com/en/github/working-with-github-pages/creating-a-github-pages-site-with-jekyll.

I Don't See Any Problem Here!


Well, it is true that mostly you won't have any issues with your content being piped through Jekyll before it gets copied to the GitHub's CDN servers..

..Unless you want to:

  1. Publish folders named node_modules or vendor (under the project's root /).

  2. Having folders with a name that starts with underscore _, full-stop . or hashtag #.

  3. Having any file or folder with a name that ends with tilde ~



You can however "force-include" those, by adding and editing your configuration file. But who wants that?..

Do I Need Jekyll?


If you don't use the default template suggested by GitHub,
and don't base your website on the README.md file,
then you don't need Jekyll!

Even if you're using the default template,
but you don't update your website - you can download your website using https://www.httrack.com/, https://en.wikipedia.org/wiki/Wget, or even with your browser's built-in "Save-As.." by pressing the CTRL+S key-combination.
Then you can clear up the repository from the old-files and upload those instead.
And again: You don't Need Jekyll!

Alright, I Don't Need Jekyll!


Very well. Include a .nojekyll file in the root of your branch that is used for the website.
Normally it's your master branch by default, although if you have an old website, you might be using the gh-pages branch, which you can also use for your website if you create it manually.
See in the settings section of your repository.

Commit. Push. That's it!
