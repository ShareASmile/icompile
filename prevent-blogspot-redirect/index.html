Solved: Sticky BlogSpot Domain - Prevent Gio-Location Redirection

Posted at Thu, 01 Dec 2016 21:40:54 GMT

I've (for example) you browse the URL https://googleblog.blogspot.com/2009/11/releasing-chromium-os-open-source.html from the UK you might be surprised to find-out that you'll be redirected to a UK-related URL: https://googleblog.blogspot.co.uk/2009/11/releasing-chromium-os-open-source.html.

Reasons aside, this might not be optimal for you for various reasons, or maybe, it simply annoys you :/

What you want is a "sticky" domain, of your choice (i.e. keep is .com or even .ru or .co.il if you feel like it..).

Well, you need to activate something that called "No Country Redirect", or NCR for short,
It is a completely server-side feature that returns (via Set-Cookie HTTP-header) a HTTP cookie (meaning you can not set it with JavaScript) setting your domain to be sticky, preventing redirection.

How to do it?

add /ncr/ as the first URL-path element, for example:
https://googleblog.blogspot.com/ncr/2009/11/releasing-chromium-os-open-source.html[ENTER]

it will then redirect (ONCE) to https://googleblog.blogspot.com/2009/11/releasing-chromium-os-open-source.html and will write a one-day old cookie named NCR with the value 1, to the domain *.googleblog.blogspot.com.

you may also use any other domain you wish to "make sticky".

if you have some favorite 'live' testing page, this little trick will help you make the test page 'more consistent', which is a good thing (although a better thing is, naturally, never to relay on real online web-pages in testing..)
