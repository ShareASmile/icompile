Google "Secret" Domain Check API Service

Posted at Thu, 06 Oct 2016 22:05:46 GMT


It is keyless, so one may query it quite a lot,
although it WILL drag-with-it every cookie you have on the same Google domain, so be cautious if you're going to try it from the same-browser you normally read emails from gmail or access other Google related services (YouTube..)

You may wrap it will a PHP/server-side service or use a plain-cURL script in which you have a full control on the sent-data.



Since I'm doing every request from me PC out in Tel-Aviv, Israel- the domain's data will surely include the .co.il Google-variation. You may encounter other variations such as .co.uk, .ru or .cn if you are from China just kidding, there is no Google in China :/







  • OpenSearch Format (probably the least known variation..)
    https://www.google.com/searchdomaincheck?format=opensearch&type=chrome
    <?xml version="1.0" encoding="UTF-8"?><OpenSearchDescription xmlns="http://a9.com/-/spec/opensearch/1.1/"><ShortName>Google</ShortName><Description>Google Search</Description><Image height="16" width="16" type="image/icon">https://www.google.co.il/favicon.ico</Image><Url rel="self" type="application/opensearchdescription+xml" template="https://www.google.com/searchdomaincheck?format=opensearch" /><Url rel="suggestions" type="application/x-suggestions+json" template="https://www.google.co.il/complete/search?q={searchTerms}&amp;inputencoding={inputEncoding}&amp;outputencoding={outputEncoding}" /><Url type="text/html" xmlns:referrer="http://a9.com/-/opensearch/extensions/referrer/1.0/" template="https://www.google.co.il/search?q={searchTerms}&amp;ie={inputEncoding?}&amp;oe={outputEncoding?}" /></OpenSearchDescription>



  • Fully Qualified URL
    https://www.google.com/searchdomaincheck?format=url&type=chrome
    https://www.google.co.il/



  • Domain Only (not even with the www prefix)
    https://www.google.com/searchdomaincheck?format=domain&type=chrome
    .google.co.il





Similar to WhatsAPP's Status API see https://icompile.eladkarako.com/whatsapps-status-api/,
It is a HTTPS (ssl) or HTTP base service which may prove to be useful to you if you wish to check internet-connection, without maintaining the server yourself, allowing text/xml; charset=UTF-8 or plain text response. It still obey the SAMEORIGIN and CORS rules, so either run it from Google's domain (for example for a Chrome extension that runs on those domains) or wrap it with a secondary server-side service.
